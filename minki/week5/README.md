### 1. S3 Multipart Upload
- **동작 방식**
  - 파일을 여러 청크로 나누어 병렬로 업로드
  - 각 청크는 독립적으로 업로드 가능
  - 모든 청크가 업로드 완료되면 최종 파일 생성

- **장점**
  - 대용량 파일 업로드 최적화
  - 네트워크 오류 발생 시 재시도 용이
  - 병렬 처리로 업로드 속도 향상

### 2. Sync Service
- **Sync Fast**
  - **Adaptive Polling**
    - 사용자 활동에 따라 폴링 주기 조정
    - 활동이 많을 때는 짧은 주기, 적을 때는 긴 주기
  - **Websocket Connection**
    - 실시간 변경사항 전송
    - 서버와 클라이언트 간 지속적인 연결 유지
    - 과도한 오버헤드
  - **Delta Sync**
    - 변경된 부분만 전송하여 네트워크 대역폭 절약
    - 청크 단위로 파일 분할 및 동기화-
